## 90초 시연 시나리오

### 사전 준비 (시연 시작 30분 전)

1. 두 URL 모두 브라우저에서 한 번씩 접속 → 슬립 해제
2. 점검 본체 URL 은 `?demo=quick&space=화학실` 로 미리 진입
3. 대시보드 URL 은 정책 우선순위 탭까지 미리 클릭해 두기

### 시연 흐름 (총 90초)

```
[0~15초]   대시보드 — 전국 11,843개교 분석 + 즉시 점검 526곳
   "교육 공공데이터 11,843개교를 분석해 즉시 점검이 필요한 526개교를 추렸습니다"
   → 시도별 현황 탭 → 정책 우선순위 탭 (1초)

[15~30초]  점검 본체 — 시연 모드 자동 진입 (?demo=quick)
   "이제 한 학교의 화학실을 점검합니다"
   → '시연 시작 (화학실)' 버튼 클릭

[30~50초]  AI 점검 — 7컷 더미 사진 자동 채움 + AI 분석
   "AI 가 사진을 분석해 흄후드·시약장·비상샤워 등 33개 설비를 탐지"
   → 자동으로 점검표(review) 단계까지 진입
   → 점검표 33개 항목 스크롤 시연

[50~70초]  Stage 2 정정 + 점검표 자동 재구성 (선택, 시간 있으면)
   "AI 가 잘못 탐지한 게 있으면 사용자가 정정 가능"
   → 1~2개 체크 → '반영하기' → 점검표 자동 업데이트

[70~90초]  대시보드 — 환원 데이터 + 정책 우선순위 탭
   "점검 결과가 다시 공공데이터로 환원되어 다음 분기 정책 정확도 향상"
   → 환원 데이터 28건 표시 → 정책 우선순위 탭의 '다음 분기 우선 100곳' 시연
   → CSV 다운로드 버튼 클릭 (실제 다운 안 해도 됨)

[마무리]  "SafeLoop = 데이터 순환형 AI 안전 플랫폼" (포지셔닝 한 문장)
```

## 돌발 상황 대처

| 상황 | 대처 |
|---|---|
| 앱이 잠자기 (sleep) 상태 | URL 방문 후 30~60초 대기 → "Yes, get this app back up!" 클릭 |
| 인터넷 끊김 | 미리 다운받은 스크린샷 PNG/PDF 로 시연 |
| AI 분석 느림 | 시연 모드는 합성 응답 (API 호출 0) → 정상 1~2초 이내 |
| 시연 모드 안 켜짐 | 사이드바 'DEMO' 배지 확인 → 없으면 [설정] → 시연 모드 ON |
| 점검표 안 보임 | 페이지 상단 '점검표 바로 보기' 큐 클릭 |

## E2E 체크리스트 (시연 1시간 전 점검)

- [ ] 점검 본체 URL 접속 → 슬립 해제
- [ ] `?demo=quick&space=화학실` 진입 → 자동 시작 안내 표시
- [ ] '시연 시작 (화학실)' 클릭 → 점검표 33개 항목 자동 표시
- [ ] '반영하기' 클릭 → review 자동 진입
- [ ] 대시보드 URL 접속 → 5개 탭 모두 로드
- [ ] 정책 우선순위 탭 → '다음 분기 우선 100곳' 표 표시
- [ ] 정책 우선순위 탭 → CSV 다운로드 버튼 작동
- [ ] 환원 데이터 28건 표시 확인
- [ ] PDF/Excel 다운로드 1회 시도 (PPT 첨부 가능 확인)
