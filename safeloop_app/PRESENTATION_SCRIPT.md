## 🎤 예상 질문 + 답변

### Q1. AI 가 정말 사진에서 안전 설비를 탐지하나요?
> 네 — Anthropic Claude / Google Gemini / OpenAI GPT 비전 모델 중 선택 가능. 시연 모드에서는 더미 이미지를 쓰므로 LAW_BASIS 기반 합성 응답이 표시되며, 화면에 **노란 배너로 명시**합니다 (정직한 데이터 출처).

### Q2. 점수가 자기보고인데 학교가 만점만 받지 않나요?
> 두 단계 검증으로 차단합니다.
> ① **자기보고 vs AI 탐지 불일치 경고** — 사용자가 "양호" 답한 항목 중 AI 가 "부재" 분류한 설비가 있으면 빨간 경고 + 보완 사진 재분석 권장.
> ② **재점검 주기 알림** — 마지막 점검 후 90일·180일 단계별 강조.

### Q3. 교육청과 SafeLoop 가 어떻게 연결되나요?
> 두 가지 방법.
> ① **앱 다이렉트 발송** (권장) — 1클릭, 수신 확인 자동 추적. 단일 PC / 같은 NAS / 같은 클라우드 인스턴스 환경.
> ② **다운로드 후 직접 발송** — PDF + .safeloop 다운 → Gmail·Naver·Daum·카톡 자유 선택.
> SafeLoop 는 메일 서버·카톡 봇 X — 외부 채널 + 교육청 수신함 수동 업로드.

### Q4. 분산 환경 (학교 PC ≠ 교육청 PC) 에서는?
> 정식 출시 시 백엔드 서버로 확장 검토. 현재 MVP 는 단일 인스턴스 / 공유 NAS / 같은 Streamlit Cloud URL 환경 한정.

### Q5. 개인정보 보호는?
> AES-256-GCM 자동 암호화 (.safeloop). 학교 식별 정보 포함되지만 SafeLoop 앱 안에서만 복호화. 메일·카톡 잘못 발송 시에도 외부인은 텍스트 에디터로 못 열음. 운영 시에는 환경변수 키 강제 (`SAFELOOP_KEY`).

### Q6. 모바일에서 동작하나요?
> 핵심 사용 환경 — 폰 카메라로 7컷 촬영. iPhone Safari·Android Chrome 호환. HEIC 파일 업로드 자동 안내. 모바일 ↔ PC 라운드트립 (.safeloop 카톡 공유 후 PC에서 자동 복호화) 검증 완료.

### Q7. 11,929개 학교 데이터는 어디서?
> 공공데이터포털 (data.go.kr) 의 학교 위험도 분석 데이터셋. SafeLoop 가 시작점으로 사용. 점검 결과는 익명화하여 공공데이터로 환원 → 다음 분기 위험도 분석 정교화에 기여 (순환 구조).
