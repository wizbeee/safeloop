## ⚠️ 주의·알아두면 좋은 점

### 코드 구조
- `LAW_BASIS` (modules/laws.py) — **36개 표준 항목 + applicable_spaces 메타** 가 모든 점수·추천·매핑의 토대
- `SHOTS` (pages/2_AI점검.py) — 9개 키 (entrance_diag · front_view · center_window/corridor/front_door/back_door · ceiling · back_door_diag · close_supplement)
- `WIZARD_STEPS` — 11단계 (shoot_1~shoot_8 + ai_run + supplement + review)
- 자동재생: 화학실/일반교실/미술실 토글 — 일반교실·미술실은 사진 없으면 chemistry_lab 폴백
- `_MIN_REQUIRED_FILLED = 5` — 필수 7컷 중 5개만 차도 분석 진행 가능 (시연 편의)

### Stage 1 제거의 영향
- AI 호출 횟수: 3회 → **2회** (Stage 2 + Stage 3)
- `_skipped: True` 플래그가 있는 stage1_result 합성됨
- "공간 유형 식별" UI 섹션은 "✓ 담당자 등록 정보" 카드로 대체
- cross_check 토글 deprecated (설정 페이지에 안내만 표시)
- 캐시 폴백: stage1 캐시 검사 → stage2 캐시(`{img_hash}_{space_type}`) 정확 매칭

### 자동 매핑 정확도 한계
- 현재 자동 매핑이 53% 수준 (39개 중 21개) — AI 가 생성한 점검표 title 에서 표준 설비명 부분 매칭
- AI 가 "흄후드 작동 상태 점검" 처럼 표현하면 매칭됨, "국소배기 시스템" 처럼 다른 용어 쓰면 누락
- **수동 매핑 expander** 가 사용자 개입 경로 — 사용자가 selectbox 로 표준 설비 지정
- LAW_BASIS 항목명에 별칭(aliases) 사전 추가 시 매핑률 향상 가능

### Streamlit Cloud 배포 시 주의
- AI 캐시 (`school_storage/_ai_cache/`) 는 컨테이너 재시작 시 초기화 → 시연 캐시 휘발 위험
- `st.session_state` 만으로 운영하려면 사용자가 매번 다운로드해 백업 필요
- `pypdf` 가 requirements.txt 에 추가됨 (통합 PDF 결합용)
