## 1. 핵심 메시지·슬로건

### 🎯 한 줄 (8 단어 이내)
> **"공공데이터로 시작해, 공공데이터로 돌아옵니다."**

### 🎯 세 줄 (포스터·발표 표지용)
> 1. 학교 안전 점검을 **종이에서 데이터로**
> 2. AI 가 사진 7컷으로 안전 설비를 식별하고 맞춤 점검표를 만듭니다
> 3. 점검 결과는 다시 공공데이터로 환원되어 다음 점검을 정교하게 합니다

### 🎯 다섯 줄 (캠페인·소개 페이지용)
> 1. **SafeLoop** — 학교 안전 순환 시스템
> 2. 11,929개 전국 학교 공공데이터에서 위험군 526개교를 식별
> 3. AI 비전(Anthropic Claude·Google Gemini·OpenAI GPT) 으로 안전 설비 자동 탐지
> 4. 학교는 점수·PDF 보고서를 받고, 교육청은 모든 학교 데이터를 자동 통합·집계
> 5. 정책 시뮬레이션 → 다음 분기 환원 → 다음 점검 정교화 — 자가 개선 순환

## 3. 한국 학교 안전 점검 현황·법령

### 법적 근거 (실제 한국 법령 — 그대로 인용 가능)

| 법령 | 조항 | 내용 |
|---|---|---|
| **교육시설법** | 제10조 제3항 | 학교 자체 안전·유지관리 점검 의무 |
| **교육시설법 시행규칙** | 제47조 | 실험실 비상샤워·세안기 설치 기준 |
| **학교보건법** | 제6조·제9조 | 학교 환경 위생·응급처치 |
| **학교안전법** | 제8조 | 안전교육·게시 의무 |
| **소방시설법** | 제11조·별표 4 | 특정소방대상물 소화설비 기준 |
| **산업안전보건법** | 제38조·제114조 | 화학물질·개인보호구·MSDS |
| **위험물안전관리법** | 제5조·제17조 | 가스 차단·고압가스 보관 |

### 현장 문제 (발표 도입부에 인용 가능)

> 1. **종이 체크리스트의 한계** — 분실, 보관 어려움, 학교마다 다른 양식
> 2. **결과 통합 부재** — 교육청은 학교별 점검 결과를 일일이 받아 분류·집계해야 함
> 3. **정책 반영 지연** — 점검 → 결재 → 발송 → 통합 → 정책 결정 사이 데이터 손실
> 4. **재점검 누락** — 한 번 점검 후 잊혀지는 사례 다수
> 5. **AI 활용 부재** — 안전 설비 탐지·맞춤 점검표는 AI 가 잘 할 수 있는 영역인데 미활용

### 정량 통계 (인용 가능 — 공공데이터 기반)
- 전국 학교 수: 약 11,929개 (정보공시)
- 시도 교육청: 17개
- 위험도 상위 10% 식별 시: 약 1,200개 학교
- 고위험 (집중 관리 필요) 식별 시: 약 526개 학교

## 4. 기존 방식 vs SafeLoop 비교

| 항목 | 기존 방식 (종이·엑셀) | SafeLoop |
|---|---|---|
| **점검표 양식** | 학교마다 다름 / 통일성 부재 | 공간별 AI 자동 생성 (LAW_BASIS 36개 표준) |
| **사진 활용** | 첨부 안 함 또는 분리 보관 | 7컷 → AI 자동 설비 탐지 |
| **점수 계산** | 수기·엑셀 / 가중치 임의 | 법령 가중치 + 자동 계산 |
| **검증 장치** | 자기보고만 / 검증 없음 | AI 탐지 vs 자기보고 **불일치 경고** |
| **결재 첨부** | PDF 별도 작성 | 통합 PDF 자동 생성 |
| **교육청 발송** | 메일·공문 / 양식 다양 | 앱 다이렉트 또는 4채널 (Gmail·Naver·Daum·카톡) |
| **수신 확인** | 학교가 일일이 확인 | 자동 추적 (✅ 수신 확인 완료) |
| **교육청 집계** | 엑셀 수작업 / 통일성 부재 | 자동 통합 + 시도별 분류 |
| **재점검 알림** | 사람이 기억 | 60·90·180일 자동 단계 알림 |
| **정책 시뮬레이션** | 별도 분석 / 즉시 반영 어려움 | 환원 데이터 기반 즉시 시뮬 |
| **데이터 보안** | 평문 파일 / 분실 위험 | AES-256-GCM 자동 암호화 |
| **모바일 지원** | X (PC 입력 후 출력) | O (폰 카메라로 즉시 점검) |

## 5. 사용자 페르소나 2명

### 👨‍🏫 페르소나 A — 김교사 (학교 안전 담당자)
- **소속**: 충남삼성고등학교 화학 교사 (3년차)
- **역할**: 학년 부장 + 화학실·과학준비실 안전 관리 담당
- **연령**: 33세
- **기술 친화도**: 중간 (스마트폰 일상 사용, 새로운 앱 학습 가능)

**현재 고민**:
- 종이 점검표가 매번 다른 양식 — "지난번엔 어떻게 작성했지?"
- 화학실 안전 설비를 다 점검하려면 30분+ 소요
- 결재 → 교육청 발송까지 사진·PDF·엑셀 따로 만들어야 함
- "내가 보낸 거 잘 도착했나?" 확인 불가

**SafeLoop 사용 시 변화**:
- 폰으로 7컷 찍고 AI 가 설비 자동 탐지 → 5분으로 단축
- 통합 PDF + 암호화 데이터 자동 생성 → 결재 첨부 즉시
- 교육청 다이렉트 발송 → "✅ 수신 확인 완료" 자동 표시
- 90일 후 자동 재점검 알림

**사용 빈도**: 분기 1회 (분기당 화학실·물리실·일반교실 3개 점검)

---

### 👩‍💼 페르소나 B — 이장학사 (교육청 시설안전 담당)
- **소속**: 충청남도교육청 시설안전과
- **역할**: 도내 600여 학교 안전 점검 결과 통합·정책 제안
- **연령**: 41세
- **기술 친화도**: 중상 (엑셀 능숙, 데이터 시각화 익숙)

**현재 고민**:
- 600개 학교가 각자 다른 양식으로 점검 결과 발송
- 메일 첨부파일을 일일이 열어 엑셀 통합 — 분기마다 1주 소요
- 위험 학교 식별이 늦어 사고 후에야 알게 됨
- 정책 의사결정에 데이터를 활용하려 해도 정량 근거 부족

**SafeLoop 사용 시 변화**:
- 학교가 발송한 .safeloop 자동 복호화 → 시도별 자동 분류
- 미열람 ●47 같은 빠른 인지 → 1시간 안에 47건 처리 가능
- 위험도 우선 정렬 → 즉시 고위험 학교 식별
- 정책 시뮬레이터 → "시도 시범 100교 약 15억" 즉시 ROI 추정

**사용 빈도**: 매주 1회 (수신 확인) + 분기 1회 (정책 보고서 작성)

## 8. 슬라이드 12장 구성안

**총 12분 발표 (1장당 60초) — 발표용 PPT 골격**

| # | 슬라이드 제목 | 핵심 내용 | 시각 자료 |
|:-:|---|---|---|
| 1 | **표지** — SafeLoop 학교 안전 순환 시스템 | 슬로건 1줄 + 발표자·소속 | SAFELOOP 로고 |
| 2 | **문제 정의** | 종이 체크리스트의 5가지 한계 + 정량 통계 | 종이↔디지털 대비 |
| 3 | **법적 근거** | 교육시설법 제10조 3항 + 6개 관련 법령 | 법령 그래프 |
| 4 | **SafeLoop 한 장 요약** | 4 Stage 도식 + 순환 구조 | Sankey 다이어그램 |
| 5 | **학교 모드 — 점검 흐름** | 점검 시작 → AI 탐지 → 점수 → 발송 | 화면 캡처 4장 |
| 6 | **AI 비전 기술** | 7컷 → 36개 안전 설비 자동 탐지 | 사진 + 탐지 결과 |
| 7 | **자기보고 검증** | AI 탐지 vs 사용자 입력 불일치 경고 | 빨간 경고 화면 |
| 8 | **교육청 모드 — 통합 분석** | 수신함 100건 처리 효율 + 이메일 클라이언트 패턴 | 수신함 화면 |

## 11. 보도자료 골격

### 헤드라인 (1줄)
> **충남삼성고 화학교사가 만든 'SafeLoop' — AI 가 학교 안전 점검을 5분으로 단축**

### 리드 (한 문단)
> 충남삼성고 화학교사 홍권능이 개발한 '**SafeLoop**' 는 학교 자체 안전 점검을 종이에서 디지털로 전환하는 시스템이다. 폰 카메라로 7컷만 찍으면 AI 가 안전 설비를 자동 탐지하고 맞춤 점검표를 생성하며, 교육청은 모든 학교 데이터를 자동 통합·집계할 수 있다. 11,929개 전국 학교 공공데이터를 시작점으로, 점검 결과를 다시 공공데이터로 환원해 정책 결정에 즉시 반영하는 자가 개선 순환 구조가 특징이다.

### 본문 (4문단)

**1문단 — 문제**: 한국 학교는 교육시설법 제10조 3항에 따라 자체 안전 점검 의무가 있다. 그러나 현장은 종이 체크리스트 분실, 학교마다 다른 양식, 교육청 통합 데이터 부재 등으로 정책 반영이 지연돼 왔다.

**2문단 — 해결**: SafeLoop 는 AI 비전(Anthropic Claude·Google Gemini·OpenAI GPT) 으로 사진에서 36개 표준 안전 설비를 자동 탐지하고, 6개 관련 법령(교육시설법·소방시설법·산업안전보건법 등) 가중치로 안전 점수를 계산한다. 자기보고와 AI 탐지가 불일치하면 빨간 경고가 떠 만점 처리를 막는다.

**3문단 — 임팩트**: 학교 1곳당 점검 시간 85분 → 8분 (90% 절감). 시도 교육청 통합 작업 64시간 → 4시간 (94% 절감). 100건 이상 처리 효율을 위해 이메일 클라이언트 패턴(미열람 우선·시간 그룹·별표·일괄 액션)을 적용했다.

**4문단 — 향후**: 단기적으로 본교 활용 + 1~3개 학교 베타 도입, 중기적으로 시도 교육청 1곳 도입, 장기적으로 전국 17개 시도 확산 + 모바일 앱·KEIIS 연동을 검토한다. AES-256-GCM 자동 암호화로 보안을 확보했다.

### 인용문
> "교사가 만든 도구라 교사 입장에서 무엇이 진짜 필요한지 알았다. 양식 통일·교육청 통합·정책 반영이 핵심이고, 종이로는 절대 안 되는 영역이다."
> — 홍권능 (개발자, 충남삼성고 화학교사)
