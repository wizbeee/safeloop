## 1. 시스템 컨텍스트

```

│  Stage 1 — 공공데이터포털 (data.go.kr)                          │

                                          ↓
┌───────────────────────────────────────────────────────────────┐

│  └──────────┘  └──────────┘  └──────────┘  └──────────┘       │

│                                                                 │
│  ┌────────────────────────────────────────────────────┐       │

│  · KEIIS 입력용 JSON 다운로드                                     │

                                  ↓
┌───────────────────────────────────────────────────────────────┐
```

## 2. 컴포넌트

## 3. 데이터 흐름

### 3.1 사진 → AI 파이프라인

```
사용자 카메라 (모바일)
    ↓ st.file_uploader (capture=environment)
captured photo bytes
    ↓ image_quality.analyze_and_optimize
검사 결과 (블러/어둠 OK?) + 회전/리사이즈된 JPEG (≤1568px)
    ↓ 사용자 확인
ai_providers.get_provider() 선택
    ↓ Anthropic Claude or OpenAI GPT-4o
단계 1: 공간 유형 + 신뢰도 + 근거
    ↓ (옵션) 교차검증으로 합의 확인
단계 2: 설비 탐지 (탐지/부재/모호) + image_ref 매핑
    ↓ 사용자 확정 (체크박스 수정 → AI 재학습 데이터)
단계 3: 맞춤 점검표 (법령 근거 병기)
```

### 3.2 점검 결과 → 4분기 저장

```
세션 상태 (st.session_state)
    ↓ storage.save_inspection
school_storage/<코드>/<세션>/
    ├─ master.json                ← 원본
    ├─ edu_package.json           ← 교육청 발송용 (식별 유지)
    ├─ opendata_package.json      ← 공공 환원용 (익명화)
    ├─ 점검결과보고서.pdf          ← 사람이 읽는 형태
    ├─ 점검결과.xlsx
    ├─ 점검결과.csv
    └─ 에듀파인_품의서.pdf

  ↓ "에듀파인 ZIP 생성" 버튼
zipfile (공문 + 모든 첨부)
  ↓ 사용자가 에듀파인에 수동 업로드 → 결재
edu_app_sent (결재 완료 체크 시)
  ↓ storage.send_to_edu_app
mock_edu_receipt/<시도>/<timestamp>_<코드>_<세션>.json
```

### 3.3 보안·익명화

```
원본 (학교명·점검자·사진) — 학교 클라우드만
    ↓ 교육청 발송 (식별 유지)
교육청 수신함
    ↓ KEIIS 입력 (식별 유지)
KEIIS DB
    ↓ 익명화·집계 (data_loader.anonymize_code)
공공데이터포털 (학교명 → ANON-XXXXX 해시, 좌표 → 시도 단위)
    ↓ 자동 환원
대시보드 B (전국 공개)
```

## 4. 외부 통합

| 통합 | 현재 상태 | Phase 2 | Phase 3 |
|---|---|---|---|
| Anthropic API | ✅ 직접 | ✅ | ✅ |
| OpenAI API | ✅ 직접 | ✅ | ✅ |
| 에듀파인 | ZIP 다운로드 (수동 업로드) | 결재 완료 PDF 자동 파싱 | 결재 이벤트 webhook |
| KEIIS | JSON 다운로드 (수동 입력) | 교육청 수동 이관 지원 | API 직접 연동 |
| 공공데이터포털 | data.go.kr CSV 정적 사용 | 분기 환원 신청 | 자동 등록 API |
| 국가법령정보센터 | 법령 매핑 정적 테이블 | RAG 부분 도입 | 전문 RAG |

## 5. 보안

| 위협 | 완화 |
|---|---|
| 공개 GitHub에 API 키 노출 | `.env` gitignore + AES-256 암호화 `.env.enc` 만 커밋 |
| 학교 인증 우회 | 학교코드 + 6자리 인증번호 (실제 운영 시 교육청 발급) |
| 사진 위변조 | EXIF 검사 + 모바일 카메라 직행 (file_uploader capture=environment) |
| 학교 식별 정보 누출 | 환원 데이터는 SHA-256 해시 + 시도 단위 집계 |
| AI 키 도난 시 비용 폭주 | 캐시 + 이미지 사전 최적화 + 공급자 한도 설정 |

## 6. 성능

| 항목 | 현재 | 목표 |
|---|---|---|
| AI 단계 1·2·3 총 소요 | 30~60초 (사진 6장 기준) | 동일 (캐시 시 즉시) |
| 동시 사용자 | ~10 (Streamlit Cloud 무료) | ~100 (유료 Tier) |
| 메모리 | 1GB 이내 | 동일 |
| 사진 한 장 평균 크기 | 3~5MB → 200~400KB (자동 리사이즈 후) | 동일 |
